<objective>
Extend the Rust API to serve symbol-level data, typed edges, and dead code flags in the /api/graph response.

Purpose: All Phase 5 browser features (expand, overlays, filters, blast radius) require symbol-level data and edge type information that the current file-level-only response does not include. This plan creates the data contract that all subsequent plans depend on.

Output: `EnrichedGraphResponse` replaces `FileGraphResponse` in the API, serving file nodes + symbol nodes + typed edges + dead code flags in a single payload (per D-81).
</objective>

Add `enriched_projection` function below `file_level_projection` (around line 232). Signature:

```rust
pub fn enriched_projection(
    graph: &CodeGraph,
    dead_result: &DeadCodeResult,
    stats: ScanStats,
    project_name: String,
) -> EnrichedGraphResponse
```

Implementation steps:
1. Call the existing file_level_projection logic internally (reuse Steps 1-6 from that function) to build `nodes: Vec<FileNode>`.
2. Build a HashSet of dead code symbol IDs from `dead_result.confirmed` and `dead_result.suspicious` for O(1) lookup.
3. Iterate `graph.graph.node_weights()` to build `symbols: Vec<SymbolNodeDto>`. For each SymbolNode:
   - Set `id` = node.id.clone()
   - Set `name` = node.name.clone()
   - Set `kind` = match node.kind { Function => "function", Class => "class", Type => "type", Interface => "interface", Hook => "hook", Enum => "enum", Module => "module" }.to_string()
   - Set `file_path` = node.file_path.clone()
   - Check if node.id is in confirmed set: is_dead_code=true, dead_code_confidence=Some("confirmed".to_string())
   - Check if node.id is in suspicious set: is_dead_code=true, dead_code_confidence=Some("suspicious".to_string())
   - Otherwise: is_dead_code=false, dead_code_confidence=None
   - Only include exported symbols (node.is_exported == true) since the browser expand shows exported symbols per VIZN-03
4. Iterate `graph.graph.edge_indices()` to build `edges: Vec<TypedEdge>`. For each edge:
   - Get (src_idx, tgt_idx) from graph.graph.edge_endpoints(edge_idx)
   - Get edge_kind from &graph.graph[edge_idx]
   - Set source = graph.graph[src_idx].file_path.clone() for file-level deduplication, but ALSO emit symbol-level edges
   - Actually, emit ALL edges at the symbol level: source = graph.graph[src_idx].id, target = graph.graph[tgt_idx].id
   - Also emit deduplicated file-level edges (same dedup as file_level_projection: skip self-loops where src_file == tgt_file, dedup via HashSet)
   - Map edge_type: EdgeKind::Import => "import", EdgeKind::Call => "call", EdgeKind::TypeRef => "type_ref", EdgeKind::ReExport => "re_export"
   - For file-level edges, use edge_type "import" (the dominant file-level relationship)

Implementation note: to avoid duplicating file_level_projection code, refactor: extract the file node + file edge logic into a shared helper, or call file_level_projection internally and decompose its result. The simplest approach: copy the file node construction from file_level_projection into enriched_projection (don't call file_level_projection since the return type differs). Keep file_level_projection intact for backward compatibility but it will no longer be called from main.rs.

```rust
#[test]
fn test_enriched_response_includes_symbols() {
    let mut graph = CodeGraph::new();
    graph.add_symbol(make_symbol("src/a.ts::foo", "src/a.ts", SymbolKind::Function, true));
    graph.add_symbol(make_symbol("src/a.ts::Bar", "src/a.ts", SymbolKind::Class, true));
    let dead_result = DeadCodeResult::default();
    let resp = enriched_projection(&graph, &dead_result, dummy_stats(), "test".to_string());
    assert_eq!(resp.symbols.len(), 2);
    assert!(resp.symbols.iter().any(|s| s.name == "foo" && s.kind == "function"));
    assert!(resp.symbols.iter().any(|s| s.name == "Bar" && s.kind == "class"));
}

#[test]
fn test_enriched_response_dead_code_flags() {
    let mut graph = CodeGraph::new();
    graph.add_symbol(make_symbol("src/a.ts::foo", "src/a.ts", SymbolKind::Function, true));
    graph.add_symbol(make_symbol("src/a.ts::bar", "src/a.ts", SymbolKind::Function, true));
    let dead_result = DeadCodeResult {
        confirmed: vec![DeadCodeEntry {
            symbol_id: "src/a.ts::foo".to_string(),
            file_path: "src/a.ts".to_string(),
            symbol_name: "foo".to_string(),
            kind: SymbolKind::Function,
            line_start: 1, line_end: 10,
            confidence: Confidence::Confirmed,
        }],
        suspicious: vec![],
    };
    let resp = enriched_projection(&graph, &dead_result, dummy_stats(), "test".to_string());
    let foo = resp.symbols.iter().find(|s| s.name == "foo").unwrap();
    assert!(foo.is_dead_code);
    assert_eq!(foo.dead_code_confidence, Some("confirmed".to_string()));
    let bar = resp.symbols.iter().find(|s| s.name == "bar").unwrap();
    assert!(!bar.is_dead_code);
    assert_eq!(bar.dead_code_confidence, None);
}

#[test]
fn test_enriched_response_typed_edges() {
    let mut graph = CodeGraph::new();
    graph.add_symbol(make_symbol("src/a.ts::foo", "src/a.ts", SymbolKind::Function, true));
    graph.add_symbol(make_symbol("src/b.ts::bar", "src/b.ts", SymbolKind::Function, true));
    graph.add_edge("src/a.ts::foo", "src/b.ts::bar", EdgeKind::Call);
    let dead_result = DeadCodeResult::default();
    let resp = enriched_projection(&graph, &dead_result, dummy_stats(), "test".to_string());
    // Symbol-level edge
    assert!(resp.edges.iter().any(|e| e.source == "src/a.ts::foo" && e.target == "src/b.ts::bar" && e.edge_type == "call"));
    // File-level edge also present
    assert!(resp.edges.iter().any(|e| e.source == "src/a.ts" && e.target == "src/b.ts"));
}
```
