The `CodeGraph` struct contains:
- `pub graph: DiGraph<SymbolNode, EdgeKind>` — the petgraph directed graph (public for algorithm access)
- `node_index: HashMap<String, NodeIndex>` — maps symbol_id to NodeIndex for O(1) lookup
- `barrel_files: HashSet<String>` — file paths of barrel files (side-channel per RESEARCH.md A2)

Implement these methods on `CodeGraph`:
- `new() -> Self` — empty graph
- `add_symbol(&mut self, node: SymbolNode) -> NodeIndex` — inserts node, updates HashMap, returns index
- `add_edge(&mut self, source_id: &str, target_id: &str, kind: EdgeKind)` — looks up both IDs in `node_index`; if both found, calls `self.graph.add_edge(src, tgt, kind)`. If either not found, silently skip (D-13 warn and continue). NEVER call `self.graph.add_edge` with unverified NodeIndex values.
- `get_index(&self, symbol_id: &str) -> Option<NodeIndex>` — HashMap lookup
- `is_barrel_file(&self, file_path: &str) -> bool` — HashSet contains check
- `mark_barrel_file(&mut self, file_path: String)` — HashSet insert
- `file_count(&self) -> usize` — count unique `file_path` values across all node weights using a HashSet
- `node_count(&self) -> usize` — delegates to `self.graph.node_count()`
- `edge_count(&self) -> usize` — delegates to `self.graph.edge_count()`

use std::path::Path;
use std::fs;
use cgraph_core::{Extractor, ExtractionResult, SymbolEdge, scan_directory};

The `Indexer` struct holds the dynamic extractor registry per D-48:
```rust
pub struct Indexer {
    extractors: Vec<Box<dyn Extractor>>,
}
```

Implement:
- `Indexer::new(extractors: Vec<Box<dyn Extractor>>) -> Self`
- `Indexer::index(&self, project_root: &Path) -> Result<CodeGraph, IndexerError>` — the main entry point:
  1. Call `scan_directory(project_root)?` to get the file list (reuses existing core logic including hidden dir / node_modules skip)
  2. Create a new `CodeGraph`
  3. Collect all extraction results: for each file in `detection.parseable`:
     a. Read file content with `fs::read_to_string(path)`. On error: `eprintln!("warn: could not read {}: {}", path.display(), e)` and `continue` (D-13)
     b. Find the first extractor where `ext.can_handle(path)` returns true
     c. If found, call `ext.extract(path, &source)` to get `ExtractionResult`
     d. Log any extraction errors: `for err in &result.errors { eprintln!("warn: {}", err); }`
     e. Add all `result.nodes` to `CodeGraph` via `code_graph.add_symbol(node)`
     f. Collect all `result.edges` into a `Vec<SymbolEdge>` for later resolution
  4. After all files are extracted, add edges to the graph: for each collected edge, call `code_graph.add_edge(&edge.source_id, &edge.target_id, edge.kind.clone())`. Note: edges with unknown targets (e.g., `unresolved::` prefixed, third-party imports like `react::useState`) will be silently dropped by `CodeGraph::add_edge`.
  5. Return the `CodeGraph`

NOTE: Barrel chain resolution and tsconfig alias resolution are NOT done in this plan. They will be added in Plan 02 (resolve.rs) which will be called between steps 3 and 4 above. For now, edges are added as-is from the extractor output.

Define the error type:
```rust
use thiserror::Error;

#[derive(Debug, Error)]
pub enum IndexerError {
    #[error("I/O error scanning {path}: {source}")]
    Io {
        path: String,
        #[source]
        source: std::io::Error,
    },
}

impl From<std::io::Error> for IndexerError {
    fn from(e: std::io::Error) -> Self {
        IndexerError::Io {
            path: String::new(),
            source: e,
        }
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> indexer | User-provided project path; untrusted file content read into memory |
| extractor output -> graph | Extractor produces symbol IDs and edge references that graph must validate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | crawl.rs (file read) | mitigate | `scan_directory` already uses `follow_links(false)` to prevent symlink traversal. Path validation (exists + is_dir) in CLI prevents escape. |
| T-03-02 | Denial of Service | crawl.rs (large file) | accept | Tree-sitter is bounded by file size; `fs::read_to_string` may OOM on very large files, but this is acceptable for a dev tool scanning source code. |
| T-03-03 | Denial of Service | graph.rs (add_edge) | mitigate | `add_edge` validates both source and target via `node_index` HashMap lookup before calling petgraph — prevents panic on unknown NodeIndex. |
</threat_model>
