<objective>
Improve tsconfig alias resolution to handle baseUrl without paths, extends chains, and multiple path targets.

Purpose: Real-world TypeScript projects (like agentcommercekit) use `baseUrl: "src"` without explicit `paths` entries, meaning `import 'lib/build-url'` should resolve to `src/lib/build-url.ts`. Many projects also use `extends: "./tsconfig.base.json"` to inherit path aliases from a parent config. Currently only the first path target is tried when multiple exist. These gaps cause import resolution misses that contribute to false-positive dead code.

Output: Enhanced TsConfigAliases in resolve.rs with baseUrl resolution, extends chain following, and multi-target path resolution.
</objective>

  <action>
  Modify `TsConfigAliases` in resolve.rs:

  **1. Enhance `load()` to follow `extends` chains:**

  Refactor into a helper `load_from_path(tsconfig_path: &Path, visited: &mut HashSet<PathBuf>) -> (HashMap<String, Vec<String>>, Option<String>)` that:
  a. Checks `visited` set for the tsconfig_path — if already visited, return empty (cycle guard)
  b. Adds tsconfig_path to visited
  c. Reads and parses the tsconfig JSON (reuse existing strip_json_comments logic)
  d. If `"extends"` field exists, recursively load the parent config by resolving the extends path relative to the current tsconfig's directory
  e. Start with parent's aliases and baseUrl, then override with child's values (child wins on conflict)
  f. Return the merged (aliases, base_url)

  The public `load(project_root)` function calls `load_from_path(project_root.join("tsconfig.json"), &mut HashSet::new())`.

  **2. Enhance `resolve()` to use baseUrl for bare specifiers:**

  Add a method `base_url_resolve(&self, raw_path: &str) -> String` or integrate into existing `resolve()`:
  - First try alias prefix matching (existing logic) — if any alias matches, return that result
  - If no alias matched AND the path does NOT start with "." or "/" (bare specifier), AND `self.base_url` is Some, prepend the baseUrl: `format!("{}/{}", base_url, raw_path)`
  - If neither applies, return raw_path unchanged

  **3. Support multiple path targets:**

  Change the internal resolution strategy: instead of `resolve()` returning the first target immediately, add a method `resolve_candidates(&self, raw_path: &str) -> Vec<String>` that returns ALL possible resolutions (one per path target, plus baseUrl variant). The existing `resolve()` method returns the first candidate (backward compat). Add `resolve_candidates()` for use by `resolve_file_path()`.

  This requires `resolve_file_path` to accept `graph: &CodeGraph` as an additional parameter. Update its signature and all call sites (only called from `resolve_edges` in the same file).
