**Dead code detection — `pub fn dead_code(graph: &CodeGraph, project_root: &Path) -> DeadCodeResult`:**

Per D-40, D-41:

Step 1: Build entry point file set. A file is an entry point if ANY of these patterns match the file's name (just the filename, not the full path — use `Path::file_name()`):
- `main.ts`, `main.tsx`, `index.ts`, `index.tsx` — BUT only at the project root (check `file_path.parent()` or check depth from project_root). Non-root index.ts files are barrels, not entry points.
- Any file in a directory named `test/`, `tests/`, `__tests__/`, or `__test__/` — check if any path component matches
- `App.tsx`, `App.ts` (React entry)
- Files matching `setup.*` or `config.*` pattern (file stem is "setup" or "config")

Step 2: For each node in the graph where `node.is_exported == true`:
- Check incoming edge count: `graph.graph.neighbors_directed(idx, Incoming).next().is_some()` — if has incoming edges, skip (not dead).
- If zero incoming edges:
  a. If file is an entry point: skip (D-40 — never flag entry point exports)
  b. If the file is a barrel file (`graph.is_barrel_file(&node.file_path)`): skip (barrel re-export sources are not "dead" — they're library organization)
  c. Check heuristic demotion (D-41 suspicious tier):
     - The symbol name as a string: for now, this heuristic check needs source text which we don't have in the graph. Instead, check if ANY other node in the graph has an edge with `target_id` containing `unresolved::` + the symbol name (approximate: if someone calls a function by name but the call edge wasn't resolved). If found, mark `Suspicious("referenced as unresolved call target")`.
     - Check if any edge in the graph has source_id pattern `file_path::<import>` with a namespace import style. Walk all edges where `kind == EdgeKind::Import`: if the edge's `source_id` file_path == some other file AND the edge's `target_id` ends with `::*` AND that file could contain a `import * as X` referencing the dead symbol's file. Simplified: if any Import edge targets `dead_symbol_file_path::*`, mark as `Suspicious("namespace import from {importing_file} could access this symbol")`.
  d. If no demotion heuristic matched: mark as `Confirmed`.

Step 3: Return `DeadCodeResult` with entries sorted by file_path for deterministic output.

Add three more public functions to `crates/indexer/src/analysis.rs`:

**1. Blast radius — `pub fn blast_radius(graph: &CodeGraph, symbol_id: &str) -> Vec<String>` (ANLS-04)**

Per D-45: use `Reversed<&DiGraph>` to walk edges backwards from the target symbol, collecting all reachable node IDs.

Implementation:
- Look up `symbol_id` in `graph.get_index()`. If None, return empty Vec.
- Create `Reversed(&graph.graph)` adaptor.
- Create `Dfs::new(reversed, start_idx)`.
- Iterate with `dfs.next(reversed)`: for each visited node that is NOT the start node, push `graph.graph[nx].id.clone()` to result.
- Return result Vec.

**2. Transitive dependencies — `pub fn transitive_deps(graph: &CodeGraph, symbol_id: &str) -> Vec<String>` (ANLS-05)**

Same as blast_radius but follows edges FORWARD (normal direction):
- Look up `symbol_id` in `graph.get_index()`. If None, return empty Vec.
- Create `Dfs::new(&graph.graph, start_idx)`.
- Iterate with `dfs.next(&graph.graph)`: for each visited node that is NOT the start node, push ID to result.
- Return result Vec.

**3. Cycle detection — `pub fn detect_cycles(graph: &CodeGraph) -> CycleResult` (ANLS-03, D-42)**

Per D-42: file-level cycle detection only. Per D-46: derive file-level view from symbol graph on demand.

Implementation:
- Build a file-level directed graph:
  a. Create `let mut file_graph: DiGraph<String, ()> = DiGraph::new();`
  b. Create `let mut file_index: HashMap<String, NodeIndex> = HashMap::new();`
  c. Iterate all nodes in `graph.graph.node_indices()`: for each node, get `node.file_path` and add to `file_graph` if not already present (using `file_index.entry().or_insert_with()`).
  d. Iterate all edges in `graph.graph.edge_indices()`: for each edge, get source and target file paths. If they differ, call `file_graph.update_edge(src_file_idx, tgt_file_idx, ())` — `update_edge` deduplicates (RESEARCH.md: use `update_edge`, NOT `add_edge` for file projection to prevent duplicate edges).
- Run Tarjan's SCC: `let sccs = tarjan_scc(&file_graph);`
- Filter to SCCs with size > 1 (size == 1 means no cycle): `.filter(|scc| scc.len() > 1)`
- Map NodeIndex back to file path strings: `.map(|scc| scc.iter().map(|&n| file_graph[n].clone()).collect())`
- Return `CycleResult { cycles }`.
