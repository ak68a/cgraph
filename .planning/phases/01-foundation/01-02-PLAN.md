**CRITICAL anti-pattern (RESEARCH.md Pitfall 3):** tree-sitter-typescript exports `LANGUAGE_TYPESCRIPT` and `LANGUAGE_TSX`, NOT `LANGUAGE`. The Swift, Go, and Python grammar crates each export `LANGUAGE`. Do not mix these patterns.

The file must contain these tests:

**1. `typescript_grammar_links_and_parses`:**
```rust
use tree_sitter::Parser;
use tree_sitter_typescript::LANGUAGE_TYPESCRIPT;

#[test]
fn typescript_grammar_links_and_parses() {
    let mut parser = Parser::new();
    parser
        .set_language(&LANGUAGE_TYPESCRIPT.into())
        .expect("Error loading TypeScript grammar");

    let source = std::fs::read_to_string("tests/fixtures/sample.ts")
        .expect("sample.ts fixture missing");
    let tree = parser.parse(&source, None).expect("parse returned None");
    let root = tree.root_node();

    assert_eq!(root.kind(), "program");
    assert!(!root.has_error(), "ERROR nodes in sample.ts");
}
```

**2. `tsx_grammar_links_and_parses`:**
- Use `tree_sitter_typescript::LANGUAGE_TSX` (NOT `LANGUAGE_TYPESCRIPT`)
- Parse `tests/fixtures/sample.tsx`
- Assert `!root.has_error()`

**3. `swift_grammar_links`:**
- Use `tree_sitter_swift::LANGUAGE`
- Call `parser.set_language(&LANGUAGE.into()).expect("Error loading Swift grammar")`
- Parse `tests/fixtures/sample.swift`
- Assert `!root.has_error()` (verify linkage AND parse, since we have a fixture)

**4. `go_grammar_links`:**
- Use `tree_sitter_go::LANGUAGE`
- Parse `tests/fixtures/sample.go`
- Assert `!root.has_error()`

**5. `python_grammar_links`:**
- Use `tree_sitter_python::LANGUAGE`
- Parse `tests/fixtures/sample.py`
- Assert `!root.has_error()`

Each test follows the same pattern: create Parser, set language, read fixture file, parse, assert no ERROR nodes. The fixture file paths are relative to the crate root (`crates/core/`), so `tests/fixtures/sample.ts` resolves correctly when running `cargo test -p cgraph-core`.
