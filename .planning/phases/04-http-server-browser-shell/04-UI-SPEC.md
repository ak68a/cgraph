## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / Tooltip | 13px | 400 | 1.5 |
| Label (node name) | 11px | 400 | 1.2 |
| Header stats | 13px | 400 | 1.0 |
| Panel heading / section | 11px | 600 | 1.2 |

Notes:
- All type renders white (#ffffff) or light gray (#a0aec0) on the dark canvas — never black.
- Node labels use 11px to stay unobtrusive beneath nodes at graph zoom levels (D-54).
- Tooltip body at 13px matches terminal/devtool conventions for the dark dev-tool aesthetic (D-50).
- No display/hero size — this is a data-dense developer tool, not a marketing page.

Source: D-50 (dark dev-tool aesthetic), D-54 (filename labels below nodes), D-55 (hover tooltip content). Sizes are defaults — CONTEXT.md did not specify px values for chrome text.

## Component Inventory

Phase 4 has no React/shadcn components. The browser shell is composed of these HTML/CSS/JS elements:

### 1. Header Bar

- Fixed top bar, full-width.
- Height: 40px.
- Background: #16213e (secondary surface).
- Left: project path or name (white, 13px).
- Right: scan stats string — "{N} files • {N} symbols • {N} edges • {N}ms" (light gray #a0aec0, 13px).
- No interactive elements in Phase 4 header.

Source: D-58.

### 2. Graph Canvas

- Fills 100vw × (100vh - 40px header).
- Background: #1a1a2e.
- SVG element managed by D3.
- No scrollbars (D3 zoom handles panning).

### 3. File Nodes

- Shape: filled circle (SVG `<circle>`).
- Radius: scales linearly — 8px at 1 export, 24px at 20+ exports. Linear interpolation: `r = 8 + (exports / 20) * 16`, capped at 24px.
- Fill: #4a9eff.
- Stroke: #2a2a4e (1px — subtle border to separate overlapping nodes).
- Label: filename only, 11px, #ffffff, positioned 6px below circle bottom edge.
- Label truncation: 20 characters. If duplicate filenames in different dirs, prepend parent dir: "utils/index.ts".

Source: D-53, D-54.

### 4. Edges

- SVG `<line>` or `<path>` with arrowhead marker.
- Stroke: #555555, opacity 0.4.
- Arrowhead: SVG marker, fill #555555, size 6×4px.
- No labels on edges in Phase 4.

Source: D-52.

### 5. Hover Tooltip

- Appears on node mouseenter, disappears on mouseleave.
- Positioned: 12px right of cursor (no viewport clipping).
- Background: #16213e, border 1px solid #2d3748, border-radius 6px.
- Padding: 8px 12px.
- Content (3 rows):
  1. Full relative path — 13px, #ffffff, weight 400.
  2. Export summary — "3 functions, 2 types" — 11px, #a0aec0, weight 400.
  3. Edge counts — "8 incoming • 4 outgoing" — 11px, #718096, weight 400.
- No interactive elements in tooltip.

Source: D-55.

### 6. Floating Legend / Control Panel

- Position: fixed, bottom-right, 32px from bottom edge, 32px from right edge.
- Width: 180px.
- Background: #16213e, border 1px solid #2d3748, border-radius 8px.
- Padding: 16px.
- Collapsed state: single row "Legend ▾" (11px, #a0aec0).
- Expanded state: color key rows + toggle.
- Color key rows: 12px colored circle + label (11px, #a0aec0), 8px vertical gap between rows.
- Toggle row: "Directory halos" label + checkbox. Default: OFF.
- Collapse/expand trigger: click anywhere on panel header row.

Source: D-59, D-56.

### 7. Directory Halos (toggle-activated)

- When ON: D3 convex hull per directory group, dashed border (#4a9eff at 0.15 opacity), fill same color at 0.05 opacity.
- When OFF: no halos rendered.
- Default: OFF.

Source: D-56.

## Force Simulation Contract

VIZN-07 requires pre-settled simulation — the graph must not jitter on load.

- Run D3 force simulation with `simulation.tick(300)` before any SVG is painted.
- `alphaDecay` tuned so 300 ticks reaches full settle for reference sizes (92 and 419 files).
- After settle: freeze simulation (`simulation.stop()`), render static SVG.
- No alpha-driven live animation on page load.
- Phase 6 (Watch Mode) will resume live simulation for incremental patch updates — do not hardcode stop behavior in a way that prevents resumption.

Source: D-50 specifics ("graph feels immediate"), VIZN-07, D-57 reference test cases.

## Layout Tuning Parameters

Reference test cases: nighthawk (92 files, ~630 edges), OversizeConnect (419 files, ~880 edges).

| D3 Force Parameter | Value | Rationale |
|--------------------|-------|-----------|
| `linkDistance` | 80px | Medium spacing between connected files |
| `chargeStrength` | -120 | Repulsion keeps nodes separated without excessive spread |
| `centerStrength` | 0.05 | Gentle pull toward viewport center |
| `collisionRadius` | node_radius + 20 | Prevents overlap at all node sizes |
| `alphaMin` for settle | 0.001 | Standard D3 settle threshold |
| `ticks` before render | 300 | Empirically sufficient for 419 nodes |

These are starting values; executor may tune after empirical measurement on reference corpora.

Source: D-57 (medium density tuning guidance), D-50 specifics (reference project sizes).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Stats header | "{N} files • {N} symbols • {N} edges • {N}ms" |
| Empty state heading | "No files found" |
| Empty state body | "cgraph found no source files in {path}. Check that the path is correct and contains .ts, .tsx, or other supported files." |
| Server start message (stdout) | "cgraph listening on http://localhost:{port} — opening browser..." |
| No-open fallback (stdout) | "cgraph listening on http://localhost:{port} — open in browser to view graph" |
| Port conflict info (stdout) | "Port {N} in use, trying {N+1}..." |
| Error state (browser) | "Failed to load graph data. Check that the cgraph server is running on http://localhost:{port}." |

Notes:
- Stats header format specified verbatim in D-58.
- Empty and error states are defaults — no upstream spec; derived from Phase 4 goal (INFR-02, VIZN-01).
- No destructive actions exist in Phase 4.

Source: D-58 (stats format), D-62 (auto-open and --no-open), D-60 (port behavior). Empty/error copy is default.

## Server Behavior Contract

| Behavior | Spec |
|----------|------|
| Default port | 3000 |
| Port conflict | Increment until available (try 3000, 3001, 3002...) |
| Auto-open command (macOS) | `open http://localhost:{port}` |
| Auto-open command (Linux) | `xdg-open http://localhost:{port}` |
| Auto-open failure | Print URL to stdout, continue |
| `--no-open` flag | Skip browser open entirely, print URL |
| API endpoint | `GET /api/graph` → JSON, file-level projection |
| Static assets | Embedded via `include_str!` or `rust-embed` |

Source: D-60, D-62.

| Dead code overlay | Phase 5 |
| Filters | Phase 5 |
| Session history | Phase 5 |

| Dark/light mode toggle | Deferred post-v1 |
| Keyboard shortcuts | Phase 5 |
